# 팀 녹음실 안정화 체크리스트

SetlistLab 팀 녹음실은 가이드 트랙, R2 녹음 파일, 파형, 믹서 상태를 함께 다루므로 로딩과 녹음 상태를 분리해서 확인해야 합니다.

## 로딩 단계

- `auth`: 로그인과 접근 권한 확인
- `guideTrack`: 가이드 트랙과 콘티 정보 조회
- `session`: 녹음 세션과 팀 권한 확인
- `recordingTracks`: 팀원 녹음 트랙 목록 조회
- `audioMetadata`: 가이드 트랙 오디오 렌더링
- `readUrls`: R2 presigned read URL 준비
- `ready`: 재생과 녹음 가능

가이드 트랙 오디오는 필수 로딩입니다. 팀원 트랙 read URL과 파형은 선택 로딩으로 취급하며, 실패해도 트랙별 재시도 버튼으로 복구할 수 있어야 합니다.

## R2 read URL 정책

- read URL은 `expiresIn`을 기준으로 `expiresAt`을 계산해 클라이언트 상태에 보관합니다.
- 만료 2분 전이면 새 URL을 발급합니다.
- 재생 fetch가 실패하면 한 번 강제 재발급 후 재시도합니다.
- signed URL, 토큰, object key는 디버그 로그에 남기지 않습니다.

## 녹음 상태

권장 상태 흐름:

- `idle`
- `requesting_permission`
- `ready`
- `recording`
- `preview`
- `uploading`
- `uploaded`
- `failed`

저장하지 않은 `preview` 녹음이 있으면 새 녹음을 바로 시작하지 않습니다. 사용자가 `다시 녹음`을 눌러 초기화하도록 안내합니다.

## 업로드 복구

- 업로드 중에는 저장 버튼을 비활성화합니다.
- 중복 업로드 요청은 lock으로 막습니다.
- 업로드 실패 시 blob이 남아 있으면 다시 저장할 수 있어야 합니다.
- 업로드 완료 후 트랙 목록과 사용량 요약을 다시 불러옵니다.

## 페이지 이탈 방지

다음 상태에서는 `beforeunload` 경고가 필요합니다.

- 녹음 중
- 업로드 중
- 저장하지 않은 미리듣기 녹음이 있음

## 모바일 안내

- 이어폰 착용 권장
- 화면 끄기/앱 전환 주의
- 녹음 전 마이크 테스트 권장
- iPhone/Safari에서는 브라우저 정책에 따라 녹음/재생이 중단될 수 있음

## QA 체크리스트

- 녹음실 진입 시 로딩바와 단계 메시지가 보이는가?
- 로딩 완료 전 재생/녹음/다운로드 버튼이 비활성화되는가?
- 가이드 트랙 준비 완료 후 버튼이 활성화되는가?
- 팀원 트랙 read URL 실패 시 트랙별 오류와 다시 불러오기 버튼이 보이는가?
- 파형 로딩 실패 시에도 재생이 가능한가?
- 첫 녹음 후 저장 전 새 녹음을 누르면 초기화 안내가 보이는가?
- 다시 녹음 시 저장하지 않은 녹음 삭제 확인이 뜨는가?
- 녹음 중 페이지를 닫으면 브라우저 경고가 뜨는가?
- 업로드 중 저장 버튼이 중복 클릭되지 않는가?
- R2 URL 만료 상황에서 강제 재발급 후 재생을 재시도하는가?
- Mute, Solo, Volume, Pan, Sync, Effects가 기존처럼 동작하는가?
- 현재 믹스 다운로드에 화면의 믹스 설정이 반영되는가?
